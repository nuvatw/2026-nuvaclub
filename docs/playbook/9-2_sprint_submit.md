# Sprint: Submit a Project

> **Section ID:** `sprint-submit`
> **Parent:** Sprint
> **Purpose:** How to submit projects
> **Access:** Solo Traveler and above

---

## TL;DR

1. Build your project → 2. Prepare assets → 3. Fill submission form → 4. Submit before deadline

---

## What You Need

### Required Fields

| Field | Description | Tips |
|-------|-------------|------|
| Title | Project name | Clear, catchy, relevant |
| Description | What it does | 2-3 paragraphs |
| Thumbnail | Cover image | 16:9 ratio, high quality |
| Tech Stack | Tools used | Be specific |

### Optional Fields

| Field | Description | Tips |
|-------|-------------|------|
| GitHub Link | Source code | Public repo preferred |
| Demo URL | Live version | Working link is gold |
| Video | Demo video | Under 3 minutes |

---

## Submission Form

### Step 1: Basic Info
- Project title
- Short tagline (one sentence)
- Category/theme fit

### Step 2: Description
- What problem does it solve?
- How does it work?
- What makes it special?

### Step 3: Media
- Upload thumbnail
- Add demo link (optional)
- Attach video (optional)

### Step 4: Technical
- List technologies used
- Add GitHub link (optional)
- Note any dependencies

### Step 5: Review & Submit
- Preview your submission
- Make final edits
- Click Submit

---

## Tips for a Great Submission

### Title
- Keep it short (3-5 words)
- Make it memorable
- Reflect what it does

**Good:** "AI Email Assistant"
**Bad:** "My Project Using GPT for Helping With Emails"

### Description

Structure it like this:

```markdown
## What It Does
[1-2 sentences explaining the problem and solution]

## How It Works
[2-3 sentences on the technical approach]

## Key Features
- Feature 1
- Feature 2
- Feature 3

## Built With
[Tech stack list]
```

### Thumbnail

| Do | Don't |
|-----|-------|
| High resolution | Blurry images |
| Show the product | Generic AI images |
| Clean design | Cluttered screenshots |
| Readable text | Tiny fonts |

### Demo Link

If possible, provide:
- Working live demo
- Clear instructions
- No login required (if possible)

---

## Common Mistakes

### Avoid These

1. **Missing deadline** - Set a reminder
2. **Bad thumbnail** - First impression matters
3. **Vague description** - Be specific
4. **Broken demo** - Test before submitting
5. **Off-theme** - Stay relevant

---

## After Submitting

### What Happens Next
1. Submission is reviewed (basic check)
2. Project goes live in library
3. Voting opens when submission closes
4. Community votes
5. Winners announced

### Can I Edit?
- Yes, until submission deadline
- After deadline: no changes
- After voting: locked

---

## Visibility Settings

### Public vs Private

| Setting | Description |
|---------|-------------|
| Public | Everyone can see |
| Private | Only you can see |

Note: Private projects don't participate in voting.

### Voyager Featured Slots
- Voyager members get featured placement
- Higher visibility during voting
- Priority in showcase

## Submission Checklist

Before hitting submit:

- [ ] Title is clear and catchy
- [ ] Description explains what/how/why
- [ ] Thumbnail looks professional
- [ ] Tech stack is listed
- [ ] Demo link works (if provided)
- [ ] Project matches theme
- [ ] Deadline hasn't passed
